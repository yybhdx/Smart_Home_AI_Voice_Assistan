# 智能家居语音交互功能设计方案

## 一、功能需求

通过语音交互实现以下功能：
- 天气查询
- 资讯播报
- 音乐播放
- 智能对话（大模型）

---

## 二、硬件方案

### 2.1 方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **STM32 + ESP32-S3** | 功能强大、可定制 | 开发复杂度高 | 推荐 |
| 离线语音模块(LD3320) | 简单、离线可用 | 只能识别固定关键词 | 简单控制 |
| 智能语音模组(天猫精灵) | 开箱即用 | 定制性差、依赖第三方 | 快速原型 |

### 2.2 推荐方案：ASR-PRO + ESP32-S3

**利用现有ASR-PRO模块 + 新增ESP32-S3模块**

| 模块 | 职责 | 优势 |
|------|------|------|
| **ASR-PRO** | 唤醒词检测、简单命令 | 离线、低功耗、响应快 |
| **ESP32-S3** | 复杂语音交互、联网 | 支持大模型、自然语言 |
| **STM32** | 主控协调、传感器 | 现有代码基础 |

### 2.3 硬件清单

| 模块 | 型号推荐 | 价格参考 | 说明 |
|------|----------|----------|------|
| **语音主控** | ESP32-S3-DevKitC | 40-60元 | 带WiFi/蓝牙，AI加速 |
| **麦克风** | INMP441 | 5-10元 | I2S数字麦克风 |
| **功放模块** | MAX98357A | 5-10元 | I2S数字功放 |
| **扬声器** | 3W/4Ω小喇叭 | 3-5元 | 输出语音 |
| **已有** | ASR-PRO开发板 | - | 唤醒词检测 |

## 四、硬件连接

### 4.1 ESP32-S3与STM32连接 (UART)

```
ESP32-S3 TX ──► STM32 RX (USART2或其他串口)
ESP32-S3 RX ◄── STM32 TX
GND ──────────── GND
```

### 4.2 ESP32-S3与麦克风 (INMP441 I2S)

```
ESP32-S3          INMP441
────────         ────────
GPIO4    ──────► SCK (时钟)
GPIO5    ──────► WS  (左右声道)
GPIO6    ◄────── SD  (数据)
3.3V     ──────► VDD
GND      ──────► GND
```

### 4.3 ESP32-S3与功放 (MAX98357A I2S)

```
ESP32-S3          MAX98357A
────────         ──────────
GPIO7    ──────► BCLK (位时钟)
GPIO8    ──────► LRC  (左右声道)
GPIO9    ──────► DIN  (数据)
5V       ──────► VDD
GND      ──────► GND
Speaker+ ──────► 扬声器正极
Speaker- ──────► 扬声器负极
```

### 4.4 ASR-PRO与STM32连接

```
ASR-PRO           STM32
────────         ──────
OUT引脚  ──────► GPIO (中断输入)
VDD      ──────► 3.3V
GND      ──────► GND
```

1. 创建百度智能云账号
2. 进入"管理控制台" -> "人工智能" -> "语音技术"
3. 创建应用，获取 `API Key` 和 `Secret Key`
4. 文心一言需要在千帆控制台单独开通

### 6.2 STM32 与 ESP32-S3 通信协议

```
命令格式: [帧头][命令码][数据长度][数据][校验]
帧头: 0xAA 0x55
命令码:
  - 0x01: 唤醒通知 (STM32 -> ESP32-S3)
  - 0x02: 传感器数据 (STM32 -> ESP32-S3)
  - 0x03: 控制命令 (ESP32-S3 -> STM32)
  - 0x04: 语音识别结果 (ESP32-S3 -> STM32)
```

## 八、注意事项

1. **ESP32-S3固件**: 确保使用支持PSRAM的固件，音频处理需要较大内存
2. **音频格式**: 百度ASR支持16kHz采样率、16bit、单声道PCM/WAV
3. **HTTPS**: ESP32-S3调用百度API需要HTTPS，注意内存占用
4. **唤醒灵敏度**: ASR-PRO需要调试唤醒词灵敏度，避免误触发
5. **功耗管理**: ESP32-S3可设置Deep Sleep，由ASR-PRO唤醒
