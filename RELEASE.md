# 智能家居 AI 语音助手 v1.0.1 (论文正式版本)

> **发布日期**: 2026年5月1日
> **版本说明**: 本版本为毕业论文最终定稿配套代码版本，涵盖了完整的硬件采集、AI 语音交互及 Android 移动端监控功能。

## v1.0.1 更新内容 (论文版本)

### 1. 核心功能实现
- **STM32F103 传感器采集**: 实现了 DHT11 (温湿度)、HC-SR501 (人体红外)、MQ-7 (CO 气体) 的稳定采集与 OLED 本地显示。
- **ESP32-S3 双核并行**: 充分利用 ESP32-S3 双核特性，Core 1 处理 AI 语音交互，Core 0 处理传感器数据解析与华为云 MQTT 上传。
- **华为云 IoTDA 集成**: 实现了设备属性上报、设备影子同步，确保数据云端持久化。
- **Android 原生 APP (Kotlin + Compose)**: 弃用 Flutter 版本，采用原生 Android 开发，通过 REST API 稳定获取华为云影子数据，解决 MQTT 连接稳定性问题。

### 2. 硬件与通信优化
- **串口通信协议**: 定义了标准 JSON 数据包格式，STM32 每秒通过 UART3 向 ESP32 发送一次完整数据。
- **报警系统**: 优化了 MQ-7 的 ADC 采样与 PPM 换算算法，设定 ADC=2000 为报警阈值，实现本地蜂鸣器与云端状态同步。
- **引脚冲突修复**: 针对小智 AI 占用 I2S 引脚的问题，重新分配了 STM32 通信串口为 GPIO8/9 (UART1)。

### 3. 论文相关标注
- 本版本代码已与论文《基于 ESP32-S3 的智能家居 AI 语音助手设计与实现》中的描述完全一致。
- 包含完整的电路接线说明、云平台配置参数以及 Android APP 源码。

## 快速开始
1. **STM32**: 使用 Keil 烧录 `Smart_home_STM32` 工程。
2. **ESP32**: 将 `ESP32-S3配置文件/sdkconfig` 复制到 `xiaozhi-esp32` 目录，执行 `idf.py build flash`。
3. **Android**: 使用 Android Studio 打开 `smart_home_android`，修改 `HuaweiIOT.kt` 中的参数后运行。
